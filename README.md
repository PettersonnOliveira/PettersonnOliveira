## 🧩 Projetos

Quatro APIs REST em Java + Spring Boot, cada uma evoluindo em complexidade — do CRUD básico até relacionamentos de muitos-para-muitos com regras de negócio reais.

<br>

### 🗂️ [TaskFlow API](https://github.com/PettersonnOliveira/taskflow-api)
Gerenciador de tarefas com criação, atualização de status e exclusão. O ponto de partida: onde consolidei a base de uma API REST — Controller, Service, Repository e DTOs, com validação de entrada e status HTTP corretos.

`Spring Boot` `Spring Data JPA` `Bean Validation`

<br>

### 📚 [Library Management API](https://github.com/PettersonnOliveira/library-management-api)
Controle de empréstimos de uma biblioteca: usuários, livros e histórico de retiradas. Primeira vez lidando com relacionamento entre entidades (`@ManyToOne`) e exceções customizadas com tratamento global de erros.

`JPA Relationships` `Custom Exceptions` `RestControllerAdvice`

<br>

### 🎟️ [Event Management API](https://github.com/PettersonnOliveira/event-management-api)
Gestão de eventos e inscrições, com controle de capacidade máxima por evento. Introduziu o padrão de entidade intermediária para resolver relações muitos-para-muitos, em vez de `@ManyToMany` direto — e consultas derivadas mais elaboradas (`existsBy`, `countBy`).

`N:N Relationship` `Derived Queries` `Business Rules`

<br>

### 🏋️ [Gym Management API](https://github.com/PettersonnOliveira/gym-management-api)
Gerenciamento de academia: alunos, treinos e matrículas. Projeto de repetição deliberada — mesma arquitetura dos anteriores, aplicada com mais autonomia e velocidade, num domínio novo.

`CRUD` `Validation` `Exception Handling`
